# 事故响应决策树

> 守静在 Step 6 决策时加载本文档。按严重度走对应分支。

## 总览图

```
                       ┌─────────────────┐
                       │  接收告警/报告    │
                       └────────┬────────┘
                                ↓
                  ┌──────────────────────────┐
                  │ classify-severity.py     │
                  │  P0 / P1 / P2 / P3       │
                  └──────────┬───────────────┘
                             ↓
        ┌────────────────────┼────────────────────┐
        ↓                    ↓                    ↓
   ┌─────────┐         ┌──────────┐         ┌──────────┐
   │   P0    │         │   P1     │         │  P2/P3   │
   └────┬────┘         └────┬─────┘         └────┬─────┘
        ↓                  ↓                    ↓
  [开战时频道]         [通知 oncall]        [排期处理]
  [@所有人]            [执行 runbook]       [加入 backlog]
  [5min 同步]          [15min 同步]
        ↓                  ↓
  ┌──────────────────────────────┐
  │ check-health + fetch-metrics │
  │  + fetch-logs                │
  │  + check-runbook-match       │
  └──────────────┬───────────────┘
                 ↓
  ┌──────────────────────────────┐
  │ 匹配到 runbook？             │
  └────┬──────────────────┬──────┘
    YES                   NO
     ↓                    ↓
  [执行 runbook]      [人工排查]
     ↓                    ↓
  ┌──────────────────────────────┐
  │  修复成功？                  │
  └────┬──────────────────┬──────┘
    YES                   NO
     ↓                    ↓
  [观察 30min]         [升级到下一层]
     ↓                    ↓
  [事故关闭]          [继续协调资源]
     ↓
  [post-mortem 48h]
```

## P0 分支（5min 响应）

```
[接收到 P0 告警]
   ↓
[T+0min] 立即创建飞书战时群
   ↓
[T+1min] @不盈 (oncall dev) + @信言 (通讯) + @管理层
   ↓
[T+2min] 拉监控快照 + 拉日志 + 匹配 runbook
   ↓
[T+3min] 输出影响范围（用户数 / 业务损失）
   ↓
[T+5min] 第一条进展同步（飞书卡片）
   ↓
[T+持续] 每 5min 同步进展
   ↓
[修复完成] 观察 30min 确认稳定
   ↓
[事故关闭 + 48h 内 post-mortem]
```

## P1 分支（15min 响应）

```
[接收到 P1 告警]
   ↓
[T+0min] 飞书卡片通知 oncall
   ↓
[T+5min] 拉监控 + 拉日志 + 匹配 runbook
   ↓
[T+10min] 执行 runbook（如匹配到）
   ↓
[T+15min] 第一条进展同步
   ↓
[T+持续] 每 15min 同步进展
   ↓
[修复完成] 观察 30min 确认稳定
   ↓
[事故关闭 + post-mortem（视影响）]
```

## P2 分支（1h 响应）

```
[接收到 P2 告警]
   ↓
[T+0min] 飞书卡片通知 oncall（异步）
   ↓
[T+1h] 标准排查流程
   ↓
[T+1h] 排期修复，加入 P2 任务池
   ↓
[修复完成] 关闭事故
```

## P3 分支（24h 响应）

```
[接收到 P3 告警]
   ↓
[T+0min] 加入 backlog
   ↓
[T+24h] 排期处理
   ↓
[如连续触发] 升一级到 P2
```

## 升级路径

- 5min 内 P0 无进展 → 升级到 oncall 主管
- 15min 内 P1 无进展 → 升级到 oncall 主管
- 修复失败 → 启用 backup system（如有）+ 升级到工程 VP
