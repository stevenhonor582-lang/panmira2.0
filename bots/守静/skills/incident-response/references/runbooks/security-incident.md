# Runbook: Security Incident（安全事故）

> 触发关键词：入侵 / 数据泄露 / 攻击 / 异常登录 / 越权 / DDoS
> 典型严重度：P0（数据泄露 / 入侵确认）

## 症状

- WAF 告警（SQL 注入 / XSS / 命令注入尝试）
- 异常登录（异地 / 暴力破解）
- 数据导出异常（大批量下载）
- 未授权访问
- 客户报告账号被盗

## 排查步骤

1. **确认攻击类型**：
   - 入侵（已获得系统权限）
   - 数据泄露（数据已外传）
   - 攻击中（DDoS / 暴力破解进行中）
2. **拉取证据**：
   - 完整审计日志
   - 网络流量
   - 异常 IP / UA
3. **评估影响**：
   - 受影响账号数
   - 受影响数据类型（密码 / 个人信息 / 资金）
   - 是否已外传
4. **保留现场**：不要立即清理日志 / 关闭服务

## 常见类型

| 类型 | 严重度 | 立即行动 |
|------|--------|---------|
| DDoS 攻击 | P0 | 启用 CDN 防护 + 联系 CDN 厂商 |
| 数据泄露 | P0 | 隔离受影响系统 + 通知法务 |
| 入侵确认 | P0 | 切断攻击者访问 + 全面审计 |
| 暴力破解 | P1 | 封禁 IP + 强制改密 |
| 越权访问 | P1 | 修补权限 + 审计日志 |
| 钓鱼邮件 | P2 | 隔离 + 通知用户 |

## 修复动作

### 数据泄露（最严重）

1. **立即隔离**受影响系统
2. **停止数据导出**（临时关闭 API / 导出功能）
3. **通知法务 + DPO**（GDPR / 等保合规）
4. **通知受影响用户**（24h 内）
5. **保留所有证据**（日志 / 流量 / 数据库快照）
6. **评估监管报告**义务

### 入侵

1. **切断攻击者访问**（封 IP / 吊销 token / 改密）
2. **审计所有账户**（异常登录 / 异常操作）
3. **回滚恶意改动**
4. **修补漏洞**
5. **加固监控**

### DDoS

1. **启用 CDN**（Cloudflare / 阿里云高防）
2. **限流**（按 IP / 区域）
3. **联系 ISP**清洗
4. **评估影响**（仅边缘 / 已影响业务）

## 验证

1. 攻击停止（监控 WAF / 流量）
2. 漏洞修补完成
3. 受影响账号 / 数据明确
4. 法务 / 合规确认可发布

## 升级（必经流程）

- **任何安全事故** → 立即通知法务 + DPO
- **数据泄露** → 升级 VP + 准备监管报告
- **入侵确认** → 升级 VP + 启用 IR（incident response）外援
- **涉及客户数据** → 24h 内通知客户

## 合规时间线

| 法规 | 通报时限 |
|------|---------|
| GDPR | 72h 内通报监管机构 |
| 个保法 | 立即通知 + 报告 |
| PCI-DSS | 24h 内通知支付品牌 |
| SOC 2 | 7 天内报告 |

## 注意事项

- **绝不删除日志**（事故日志归档 1 年）
- **不公开讨论细节**（仅在安全群沟通）
- **不向攻击者妥协**（不付赎金 / 不接受谈判）
- **保留所有证据链**（法务可能需要）
