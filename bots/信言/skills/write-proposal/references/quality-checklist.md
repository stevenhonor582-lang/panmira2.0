# 质量自检清单 (Quality Checklist)

> 提交方案前逐项打勾。任何一项未过都必须修正。
> 自检分四类：结构 / 内容 / 语言 / 格式。
> 配套脚本：scripts/word-counter.py / toc-generator.py 可自动验证部分项。

## 1. 结构 (Structure)

- [ ] 标题（# 唯一）与受众匹配，不超 30 字
- [ ] 必有执行摘要（管理层 / 投资人 必含，300 字内）
- [ ] 目录（## 二级标题，3-7 个章节）
- [ ] 章节顺序遵循"背景 → 方案 → 实施 → 风险"逻辑
- [ ] 必含"风险与对策"章节，未省略
- [ ] 章节标题层级清晰，不跳级（H1 → H2 → H3）
- [ ] 附录材料单独成节，与正文逻辑分离

## 2. 内容 (Content)

- [ ] 至少 3 个可量化指标（如：周期 / 成本 / 增长）
- [ ] 核心数据有来源标注（数据来源 / 假设说明）
- [ ] 目标受众的痛点在前 1/3 章节出现
- [ ] 关键术语在首次出现时给出解释
- [ ] 涉及竞品 / 第三方，措辞客观无贬损
- [ ] 关键结论（数字 / 决策）单独成段或加粗
- [ ] 方案与既定 OKR / 战略目标对齐
- [ ] key_points 中每个要点在正文都有对应段落

## 3. 语言 (Language)

- [ ] 无错别字（特别是形近字：账/帐、价/介、部署/布署）
- [ ] 避免"非常 / 十分 / 极其"等极端副词
- [ ] 主谓宾完整，无半截句
- [ ] 代词指代明确（"这 / 那"有清晰对象）
- [ ] 中英文混排：英文前后留空格
- [ ] 数字与单位之间留空格（"3 个月" 而非 "3个月"）
- [ ] 无过时引用（如"去年"改为具体年份）
- [ ] 长难句拆成 2-3 个短句

## 4. 格式 (Format)

- [ ] Markdown 渲染无破图（无未闭合代码块 / 表格）
- [ ] 关键数据用表格或列表，不堆长段落
- [ ] 链接 / 引用的 URL 有效可点击
- [ ] 列表项符号统一（`-` 或 `*`，全篇一致）
- [ ] 标题前后空一行，章节内段落不超 5 行
- [ ] 飞书上传后图片 / 表格样式正常
- [ ] 总字数在目标 length 范围（500 / 1500 / 3000 / 5000+）

## 使用方式

```bash

python3 scripts/toc-generator.py < draft.md | jq '.toc_lines'

# 2. 统计字数验证 length
python3 scripts/word-counter.py < draft.md | jq '.total_chars, .chinese_chars'

# 4. 通过则提交；不通过则修订后重检
```
