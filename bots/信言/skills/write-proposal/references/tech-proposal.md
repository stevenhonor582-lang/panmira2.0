# 技术方案模板

> 适用 topic_category: 技术方案
> 适用 audience: 技术团队 / 管理层
> 默认 length: 1500字-3000字

## 必含章节

1. 背景与挑战 (Background & Challenges)
2. 方案目标 (Goals)
3. 架构设计 (Architecture Design)
4. 关键技术选型 (Tech Stack)
5. 实施路径 (Implementation Path)
6. 风险评估 (Risk Assessment)
7. 性能与可扩展性 (Performance & Scalability)
8. 运维与监控 (Operations & Monitoring)

## 章节写作要点

### 1. 背景与挑战
- 当前系统/流程的痛点
- 技术债务或瓶颈
- 业务驱动（用户量 / 合规 / 成本）

### 2. 方案目标
- 解决 1-2 个核心问题
- 量化指标（如：P99 < 200ms, 成本降低 30%）
- 非目标（明确不做什么）

### 3. 架构设计
- 整体架构图（用文字描述模块关系）
- 核心模块职责
- 数据流（输入 / 处理 / 输出）

### 4. 关键技术选型
- 选型理由（不是"流行"，而是"为什么"）
- 备选方案及取舍
- 与现有系统的兼容性

### 5. 实施路径
- 分阶段交付（MVP → 完善 → 优化）
- 每个阶段的可验证产出
- 依赖与前置条件

### 6. 风险评估
- 技术风险（性能 / 可靠性 / 安全）
- 业务风险（迁移 / 用户影响）
- 缓解措施

### 7. 性能与可扩展性
- 预期性能指标
- 容量规划（当前 / 1 年 / 3 年）
- 扩展性设计（水平 / 垂直）

### 8. 运维与监控
- 关键监控指标
- 告警策略
- 故障恢复 (RTO / RPO)
