# 受众风格指南

> 适用所有 topic_category，按 audience 调整语气、深度、措辞
> 写提案前先确认目标读者，再选对应风格

## 1. 管理层 (Executive)

**特征**：决策者，时间稀缺，关注 ROI 与风险。

| 维度 | 建议 |
|------|------|
| 篇幅 | 1500字以内，长文加 TL;DR |
| 结构 | 结论先行：执行摘要 → 关键数字 → 决策点 |
| 措辞 | 简洁、肯定、避免技术黑话 |
| 数据 | 至少 3 个量化指标（增长 / 成本 / 周期） |
| 段落 | 短段（3-5 行），加粗关键结论 |

**禁用词**：技术细节、实现路径、源码示例
**必须含**：建议结论、所需资源、预期回报

## 2. 技术团队 (Technical)

**特征**：工程师 / 架构师，关注实现可行性、风险、性能。

| 维度 | 建议 |
|------|------|
| 篇幅 | 3000-5000字，深度优先 |
| 结构 | 问题 → 方案 → 选型 → 验证 → 风险 |
| 措辞 | 准确、严谨、给出备选方案与取舍 |
| 数据 | 性能指标、容量规划、兼容性矩阵 |
| 段落 | 可含代码片段、架构图、表格 |

**禁用词**：销售腔、过度承诺、模糊的"业界领先"
**必须含**：架构图、关键技术选型理由、监控与回滚

## 3. 客户 (Customer)

**特征**：外部付费方，关注价值、案例、风险。

| 维度 | 建议 |
|------|------|
| 篇幅 | 2000字左右，故事化 |
| 结构 | 痛点共鸣 → 解决方案 → 客户案例 → 投资回报 |
| 措辞 | 价值导向、第二人称（"您"）、避免内部术语 |
| 数据 | 行业 benchmark、ROI 测算、签约周期 |
| 段落 | 配图友好，关键句高亮 |

**禁用词**：内部代号、未经核实的承诺、竞品贬损
**必须含**：成功案例、报价与合同要点、交付周期

## 4. 投资人 (Investor)

**特征**：基金 / 战投，关注增长、市场、回报、退出。

| 维度 | 建议 |
|------|------|
| 篇幅 | 2000-3000字，附录放详细数据 |
| 结构 | 愿景 → 市场 → 打法 → 财务 → 团队 → 融资计划 |
| 措辞 | 增长叙事、里程碑、清晰假设 |
| 数据 | TAM/SAM/SOM、毛利模型、单位经济性 |
| 段落 | 数据可视化、可比公司 |

**禁用词**：模糊的"很大市场"、未经验证的数字
**必须含**：退出路径、关键里程碑、风险揭示

## 5. 合作伙伴 (Partner)

**特征**：上下游 / 渠道，关注互惠、协同、长期。

| 维度 | 建议 |
|------|------|
| 篇幅 | 1500字，强调互惠条款 |
| 结构 | 双方价值 → 合作模式 → 资源互换 → 长期愿景 |
| 措辞 | 平等、尊重、强调双方收益 |
| 数据 | 联合方案收入预测、客户共享规模 |
| 段落 | 案例研究、共建机制 |

**禁用词**：单边收益、强制条款、不对等承诺
**必须含**：合作模式图、双方 KPI、长期愿景
