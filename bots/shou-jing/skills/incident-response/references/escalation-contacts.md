# 升级联系链

> 守静在 Step 7 协调资源时加载本文档。按层级逐级升级。

## 升级链（守静 → 不盈 → 信言 → 管理层）

### 第 0 层：自动响应（守静）

- **角色**：oncall 第一响应
- **响应时间**：即时
- **职责**：
  - 故障分级
  - 拉监控 + 拉日志 + 匹配 runbook
  - 发送飞书卡片
  - 协调资源

### 第 1 层：研发响应（不盈 / bu-ying）

- **角色**：oncall 研发
- **响应时间**：P0 5min / P1 15min
- **职责**：
  - 代码修复
  - 回滚 / 部署
  - 紧急 hotfix
- **联系**：
  - 飞书 @不盈
  - 备选：飞书 oncall 群

### 第 2 层：通讯响应（信言 / xin-yan）

- **角色**：对外通讯
- **响应时间**：P0 5min 内发首条 / 持续更新
- **职责**：
  - 状态页更新
  - 客户邮件 / 短信
  - 管理层简报
- **联系**：
  - 飞书 @信言
  - 备选：comm@company.com

### 第 3 层：oncall 主管

- **角色**：研发 oncall 主管
- **响应时间**：P0 8min / P1 30min 内无进展时升级
- **职责**：
  - 协调跨团队资源
  - 决策重大变更（数据库切换 / 全量回滚）
  - 批准启用 backup system
- **联系**：
  - 飞书 oncall 主管群
  - 备选：手机（oncall 轮值表）

### 第 4 层：管理层

- **角色**：CTO / 工程 VP
- **响应时间**：P0 持续 30min 无缓解时升级
- **职责**：
  - 业务决策
  - 客户关系
  - 媒体 / 法务协调
- **联系**：
  - 飞书 VP 群
  - 备选：紧急联系电话

### 第 5 层：法务 / 合规（仅安全事件）

- **角色**：法务 / DPO
- **响应时间**：数据泄露确认后立即
- **职责**：
  - 合规通报（GDPR / 等保）
  - 客户通知
  - 监管报告
- **联系**：
  - 飞书 legal 群
  - 备选：legal@company.com

## 升级规则

| 触发条件 | 升级到 |
|---------|--------|
| P0 5min 无进展 | 第 3 层 |
| P1 15min 无进展 | 第 3 层 |
| P0 持续 30min 无缓解 | 第 4 层 |
| 检测到数据泄露 | 第 5 层（并行通知第 4 层） |
| 涉及支付 / 资金 | 第 4 层（同步） |
| 修复方案涉及重大变更 | 第 3 层审批 |

## 不可达处理

- 当前层联系人不可达 → 跳到下一层 + 飞书标记 @all
- 主管群无人响应 → 飞书 + 短信双通道
- 全部不可达 → 飞书全员群 @所有人
