# Task Router — 任务分级路由器

## 何时触发

用户提出任何需要执行的任务时，先经过本技能判断级别。

## 不要触发

- 纯问答/知识查询（直接回答）
- 闲聊（直接回复）

---

## 第一步：判断任务级别

分析用户消息，根据以下信号判断：

### L1 简单任务

信号（满足任一）：
- 单文件/单位置修改，指令明确无歧义
- "改一下"、"快速"、"帮我改"、"修个bug"
- 不涉及多模块、多技能
- 预计1分钟内完成

### L2 中等任务

信号（满足任一）：
- 需要1-3个技能协作
- 有明确交付物（报告/邮件/PPT/图片）
- "写一个"、"做一个"、"分析一下"、"生成"
- 用户意图明确，不需要深入讨论

### L3 复杂任务

信号（满足任一）：
- 需要3个以上技能协作
- 涉及架构/战略决策
- 多个不同类型交付物
- "一套"、"全套"、"从...到..."、"完整的"
- 需要深入理解用户需求后才能动手

---

## 第二步：路由到执行模式

### L1 → 直接执行

规则：
- 不调用任何流程技能
- 最多1轮确认（如果指令模糊）
- 执行后自动做基本检查
- 直接交付

输出：
```
[L1] 直接执行：[简要描述]
[执行结果]
```

### L2 → 轻量编排

规则：
- 调用 mission-planner（轻量模式）
- 不触发 brainstorming
- 自动匹配1-3个技能
- 最多1轮确认（开始前确认技能选择）
- 串行执行，每步报告进度
- 每步使用技能内建质量门禁

输出（确认时）：
```
[L2] 轻量编排，计划使用：
  Step 1: [技能名] → [产出]
  Step 2: [技能名] → [产出]
  ...
  确认执行？
```

### L3 → 完整流程

规则：
- 调用 brainstorming 理解需求（允许2-3轮问答）
- 调用 mission-planner（完整模式）拆解+匹配技能
- 展示完整执行计划，用户确认
- 逐步执行，支持并行
- 多级质检
- 最终验证+交付

输出（brainstorming后）：
```
[L3] 完整流程，理解任务如下：
  目标：...
  交付物：[列表]
  约束：...
  
  执行计划（N步）：
  Step 1: [技能] → [产出]
  Step 2: [技能] → [产出]
  ...
  确认执行？
```

---

## 第三步：用户覆盖

用户可以手动指定级别：
- "简单做" / "快速" / "L1" → 降级
- "认真做" / "详细" / "L3" → 升级

---

## 示例

### L1 示例
```
用户："把README里的项目名改成MetaBot"
路由：L1 → 直接修改
输出：[L1] 已将README中的项目名更新为MetaBot
```

### L2 示例
```
用户："帮我写一封给投资人的更新邮件"
路由：L2 → mission-planner轻量
计划：email-ops
输出：[L2] 使用 email-ops 起草投资人更新邮件
```

### L3 示例
```
用户："帮我做一套AI编程助手赛道的完整分析，要有报告和PPT"
路由：L3 → brainstorming → mission-planner完整
计划：market-research → article-writing → frontend-slides
输出：[L3] 完整流程，3步执行...
```
