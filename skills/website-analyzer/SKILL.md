# 网站运营智能评估 (Website Operations Analyzer)

## 概述

输入任意网址，自动生成专业的网站运营评估报告。覆盖 SEO、内容、技术、结构四大维度，输出评分+问题清单+优化建议。

## 触发方式

- 关键词: `分析网站`, `网站分析`, `SEO分析`, `运营评估`, `网站体检`, `网站诊断`, `analyze site`
- 直接输入 URL + 分析意图
- 命令: `/analyze-site <url> [quick|standard|deep]`

## 分析模式

| 模式 | 耗时 | 覆盖范围 | CLI 参数 |
|------|------|----------|----------|
| 快速诊断 | ~5分钟 | 首页 HTML 分析 | `--mode quick` |
| 标准审计 | ~15分钟 | 首页+crawl 20个内页 | `--mode standard` |
| 深度尽调 | ~30分钟+ | 全站爬取+策略建议 | `--mode deep` |

## 评估维度

### 1. SEO 评分 (权重 35%)
- Title 标签质量和长度 (30-60字符最佳)
- Meta Description (50-160字符)
- 标题层级结构 (H1-H6)
- Open Graph / Twitter Card
- Canonical URL
- robots.txt 和 sitemap.xml
- 结构化数据 (Schema.org)
- URL 结构规范性
- 关键词布局和密度

### 2. 内容评分 (权重 30%)
- 内容长度和丰富度 (>300字符)
- 关键词覆盖和密度
- 图片 alt 文本覆盖率
- 内外链比例
- 中文内容质量（适用时）
- 可读性和排版

### 3. 技术评分 (权重 25%)
- HTTPS 部署和证书有效期
- 安全头 (HSTS, CSP, X-Frame-Options, X-Content-Type-Options)
- 移动端适配 (viewport meta)
- 页面体积和加载速度
- 技术栈识别
- Favicon 和基础 SEO 元素

### 4. 基础信息 (权重 10%)
- 域名信息
- 服务器类型
- 页面资源统计
- SSL 证书信息

## 评分等级

| 等级 | 分数 | 含义 |
|------|------|------|
| A | 90-100 | 优秀，各项指标健康 |
| B | 75-89 | 良好，存在少量可优化项 |
| C | 60-74 | 一般，有明显改进空间 |
| D | 40-59 | 较差，需要重点整改 |
| F | 0-39 | 严重，存在重大问题 |

## 执行流程

### 快速诊断
```
1. 调用: python3 analyze.py <url> --mode quick
2. 解读 JSON 结果
3. 按「关键问题→评分→优化建议」结构输出报告到对话
4. 保存 JSON 到知识库
```

### 标准审计
```
1. 调用: python3 analyze.py <url> --mode standard
2. 解析首页+内页数据
3. 生成 Markdown 摘要到对话
4. 生成 HTML 完整报告
5. 保存 HTML 到知识库 outputs 目录
```

### 深度尽调
```
1. 调用: python3 analyze.py <url> --mode deep
2. 解析全站数据
3. 补充竞争分析建议 (基于 LLM 知识)
4. 补充内容策略建议
5. 补充变现优化建议
6. 生成完整 HTML 专业报告
7. 保存到知识库 + 发送链接
```

### HTML 完整报告

使用标准 HTML5 + 内联 CSS，包含：
- 头部: 网站截图占位、域名、评分圆圈、等级徽章
- 评分卡片: 四个维度得分+进度条
- 问题清单: 按严重程度(critical/warning/info)分类
- 关键词云/技术栈标签
- 内页分析表 (标准/深度模式)
- 优化建议: 按优先级排列
- 页脚: 分析时间、工具版本

## 专业分析视角

作为网站运营专家，需额外关注：

1. **流量变现**: 广告位布局、CTA 按钮、转化路径、付费入口
2. **内容策略**: 内容更新频率(博客/新闻)、内容类型丰富度、目标关键词覆盖
3. **用户体验**: 导航清晰度、搜索功能、面包屑导航、移动端体验
4. **竞争定位**: 相比同类型网站的差异化和优势
5. **增长机会**: 未覆盖的关键词、可以添加的内容类型、技术改进收益

## 流量-内容反馈分析（第五维度，权重 15%）

### 工作原理

通过对接 Umami Analytics API 拉取实际流量数据，将**技术体检分数**与**真实流量表现**做关联分析，识别网站的"流量密码"。

### 数据来源

| 接口 | 数据 | 用途 |
|------|------|------|
| `/api/websites/:id/stats` | PV/UV/会话/跳出率/总时长 | 流量全景 |
| `/api/websites/:id/metrics?type=url` | 各页面 PV 排名 | 内容-流量关联 |
| `/api/websites/:id/metrics?type=referrer_domain` | 引荐来源 | 流量渠道分析 |
| `/api/websites/:id/metrics?type=channel` | 渠道分类（搜索/社交/直接） | 渠道健康度 |

### 分析维度

**1. 跳出率诊断**
- >80%: critical — 内容与搜索意图不匹配，或页面加载太慢
- >60%: warning — 需要优化落地页和 CTA
- <40%: healthy — 用户体验良好

**2. 流量集中度（帕累托分析）**
- 前 3 页占总流量 >80% → 高风险，抗打击能力弱，需拓展长尾
- 分布均匀 → 健康的长尾生态

**3. 渠道多样性**
- 单一渠道 >80% → 过度依赖风险
- 多渠道分散 → 抗风险能力强

**4. 用户停留质量**
- 平均会话 <15 秒 → 内容未满足意图或加载慢
- 平均会话 >60 秒 → 内容粘性好

**5. 内容缺口发现**
- 高流量页面未在爬取范围内 → 可能是功能页（登录/支付），需单独评估
- 高质量内容页面零流量 → 排名未覆盖，需要推广或内链引流

### 评分计算

起始 100 分，扣除项：
- 跳出率 >90%：-30 | >80%：-20 | >60%：-10
- 零流量：-20 | <100PV/月：-10
- 无引荐来源：-10 | 单一来源：-5
- 每个 critical insight：-10 | 每个 warning insight：-5

### 使用方式

```bash
# 仅技术分析（不需要 Umami）
python3 analyze.py example.com

# 技术 + 流量联合分析
python3 analyze.py example.com --mode standard \
  --umami-url "https://your-server/umami" \
  --umami-user "admin" \
  --umami-pass "password" \
  --umami-site-id "website-uuid-from-umami"
```

### 流量反哺内容策略

分析报告不仅给出分数，还会生成具体的**内容行动清单**：
1. 高流量低分页面 → 优先优化 SEO 和内容质量
2. 低流量高分页面 → 需要加强内链 + 外链推广
3. 高跳出率落地页 → 检查标题/描述是否虚假承诺，优化首屏
4. 搜索占比过高 → 建议启动社交媒体 + 邮件营销多渠道策略
5. 长尾关键词机会 → 基于实际搜索词数据推荐新内容选题
