# VMT 逐模块文案生成器

## 功能边界
按照VMT模块生成操作手册的M00-M13规范，逐模块生成三层阅读结构的文案。

## 每个模块的生成流程
1. 从知识底座检索该模块所需数据
2. 按模块的结构模板(YAML)组织数据
3. 按可读性方案的三层阅读设计生成文案
4. 按去AI化规则自检

## 三层文案结构(每个模块)
```
Layer 1 (5秒扫描): 标题 + 大字突出数字 + 标签
Layer 2 (30秒略读): 结论前置句 + 关键数据 + 意义转化
Layer 3 (3分钟精读): 完整数据 + 对比 + 详情链接
```

## 各模块核心生成规则

### M00 ArticleIntro
- Hook类型选择: 痛点直击型/场景导入型/概念导入型/故事型(四选一)
- 三段式: Hook(≤60词) + Value + Promise
- 必须有时间承诺声明

### M01 Hero
- 标题: [工艺/产品名] + [限定词] + | + [品牌名] (4-10词)
- 副标题: [能力范围] + [为谁服务] + [差异化价值]
- CTA: 动词 + 时间承诺
- 信任徽章: 3-4个可验证数据

### M02 TrustSignals
- 3-5个信号覆盖≥3个维度(认证/规模/速度/客户)
- 每个信号有图标+数字+意义转化
- 按说服力排序: 认证>客户评价>Logo>数据>媒体

### M04 TechSpecs
- 参数分2-4个类别,每类3-5行
- 每个参数: 名称+值+单位+Typical/Max标注
- 每行配"vs行业标准"对比
- 总数≤12行

### M05 MaterialSelector
- 默认展示5-10种推荐材料
- 每列: 密度/强度/硬度/耐腐蚀/可加工性/应用/相对成本
- 每种有compare_note
- 可筛选(类型/性能/应用)

### M06 CaseStudy
- 挑战→方案→效果三段式
- 挑战有具体参数,效果有≥3个量化指标
- 每案例≤300字正文
- 有真实零件照片

### M07 Quality
- 四件套: 证书+检测设备+质检流程+出货报告
- 流程每步: 检测什么+用什么+频率

### M11 FAQ
- 8-15个问题,覆盖价格/交期/技术/流程/物流/起订量≥6类
- 答案: 正面回答+补充细节+行动引导
- Schema.org FAQPage标记

## 关键规则
1. 每个模块生成后立即执行去AI化自检(对照附录A)
2. 技术参数必须从R4-技术库提取,不编造
3. 所有"对你的意义"必须有数据支撑
4. 产品页总字数控制在1500-3500字
