# VMT 升级转介

## 功能边界
判断何时需要人工介入,转给正确的角色,且附带完整上下文(不让客户重复说)。

## 转介规则
```
复杂技术问题 → 人工工程师
  - 触发: 超出标准能力范围的定制需求
  - 携带: 完整图纸+客户问题+知识库检索结果
  - 提示: "客户询问是否能做0.5mm壁厚的钛合金件,需评估可行性"

报价谈判 → 商务报价师
  - 触发: 客户要求折扣/调整付款条件
  - 携带: 原始报价单+客户谈判要点+竞品价格参考
  - 提示: "客户要求降价15%,竞品报价$X,建议给8%折扣或送表面处理"

客户投诉 → 管理层
  - 触发: 质量问题/延迟交付/客户明显不满
  - 携带: 完整交互记录+问题分析+初步补救方案
  - 提示: "客户收到100件零件,其中8件公差超标"

定制需求 → 内容策略师
  - 触发: 客户需要VMT没有但在能力范围内的产品
  - 携带: 客户具体需求+技术分析+市场机会评估
  - 提示: "客户询问铝基碳化硅复合材料的加工,搜量上升可评估为新品类"
```

## 输出
```json
{
  "escalation": {
    "to": "商务报价师",
    "reason": "客户要求数量折扣(1000件以上)",
    "context": {
      "originalQuote": "...",
      "clientRequest": "15% discount for 1000+ quantity",
      "competitorReference": "竞品报价区间$8-12/pc(VMT报价$10/pc)",
      "suggestedResponse": "可考虑8%折扣+免费表面处理,底线不低$8.5/pc"
    },
    "clientShouldNotRepeat": true
  }
}
```

## 关键规则
1. 转介时附带完整上下文
2. 客户不需要重复说一遍
3. 转介后跟进确认被接收方处理
